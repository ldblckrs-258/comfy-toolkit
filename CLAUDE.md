# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Package manager is **pnpm**. The dev server is typically already running on port 3000 — do not start it.

```bash
pnpm dev                      # vite dev on http://localhost:3000
pnpm build                    # production build
pnpm test                     # run all Vitest unit tests (vitest run)
pnpm vitest run src/lib/tools/jwt.test.ts   # run a single test file
pnpm lint                     # eslint
pnpm format                   # prettier --write . && eslint --fix
pnpm check                    # prettier --check .
pnpm generate-routes          # regenerate src/routeTree.gen.ts (tsr generate)
pnpm generate-og              # regenerate public/og/*.png from the tool registry
pnpm deploy                   # build + wrangler deploy (Cloudflare Workers)
```

## Architecture

ComfyToolkit is a **TanStack Start** app (SSR + server functions) that runs on **Cloudflare Workers**. Every tool computes entirely client-side — no user data is sent to a server. UI is React 19 (with the React Compiler via `babel-plugin-react-compiler`), Tailwind CSS v4, Radix UI primitives, and cmdk for the command palette.

### The tool registry is the single source of truth

`src/lib/tools/registry.ts` defines `TOOLS` (and `ToolGroup`/`GROUP_*` metadata). This one array drives everything downstream:

- the home grid (`src/routes/index.tsx`) and sidebar
- the `⌘K` command palette (`src/lib/command-palette.ts`)
- the `sitemap.xml` server route
- the OG image generator (`scripts/generate-og.mts`)

A tool's `id` is the join key: it maps to `src/lib/tools/<id>.ts` logic, the `to` route path, and `public/og/<id>.png`.

### Two-layer tool structure

- **Logic** lives in `src/lib/tools/*.ts` as pure, framework-free functions with colocated `*.test.ts` (e.g. `hmac.ts` + `hmac.test.ts`). Crypto uses the Web Crypto API (`crypto.subtle`). This is where business logic and tests belong.
- **UI** lives in `src/routes/tools/*.tsx` as file-based routes. Each route calls `requireTool('<id>')` for metadata, builds head/SEO tags via `buildSeo`/`ogUrl`, and wires the lib functions to shared components (`Card`, `ToolHeader`, `Tabs`, `tool-panel`).

### Adding a new tool

1. Write logic + tests in `src/lib/tools/<id>.ts` and `<id>.test.ts`.
2. Add a UI route in `src/routes/tools/<id>.tsx` (copy an existing one, e.g. `base64.tsx`).
3. Add a `ToolMeta` entry to `TOOLS` in `registry.ts`.
4. Run `pnpm generate-routes`, then `pnpm generate-og` to mint the social card.

### Routing & generated files

File-based routing under `src/routes/`. `src/routeTree.gen.ts` is generated by `tsr` — **never edit it by hand**; run `pnpm generate-routes`. Server-only routes (no React component) use the `server.handlers` shape — see `src/routes/sitemap[.]xml.ts`. Router setup (TanStack Query SSR integration, `defaultPreload: 'intent'`) is in `src/router.tsx`.

### SEO & OG images

`src/lib/seo.ts` centralizes SITE constants, `buildSeo()` (meta + canonical), and `ogUrl()`. `SITE_URL` comes from `VITE_SITE_URL` (falls back to localhost). OG cards are **pre-rendered to static PNGs** at build time by `scripts/generate-og.mts` (satori + resvg, fetches Plus Jakarta Sans from Google Fonts) — they are not generated at request time. Regenerate after changing any tool's name/description/group.

### Theme & persisted state

- Theme is a `light` class toggled on `<html>` (default is dark; absence of the class = dark). `THEME_INIT_SCRIPT` from `src/lib/theme.ts` is inlined in `__root.tsx` `<head>` to apply theme before first paint and avoid flash. Persisted to `localStorage` key `comfy-toolkit-theme`.
- Per-tool input state uses `usePersistedState(key, initial)` (`src/lib/use-persisted-state.ts`), which namespaces all keys under `comfy-toolkit:`.

## Conventions

- **Path aliases:** both `@/*` and `#/*` resolve to `./src/*`. Code uses `@/` (e.g. `import { Card } from '@/components/tools/card'`).
- **Prettier:** no semicolons, single quotes, trailing commas. Run `pnpm format` before finishing.
- **Style:** prefer `Array<T>` over `T[]` (matches the codebase; the eslint `array-type` rule is disabled). `strict` TS with `noUnusedLocals`/`noUnusedParameters` on.
- Keep tool logic pure and in `src/lib/tools/`; keep React/SSR concerns in routes and components.

## Deploy

Cloudflare Workers via `wrangler.jsonc` (custom domain `comfytk.com`, `nodejs_compat`). `main` is `@tanstack/react-start/server-entry`. `pnpm deploy` builds then runs `wrangler deploy`.
